## Stackroot 0.3.6

Download and run `Stackroot-Setup-0.3.6.exe` on **Windows 10/11 (64-bit)**. Update over **0.3.5** — sites, databases, and settings are kept.

### PHP catalog

- **Install works again** — current patch releases are available: PHP **8.5.8**, **8.4.23**, **8.3.32**, and **8.2.32**. Older pinned builds had been moved by PHP.net and returned 404.
- **Previous patches kept** — the prior catalog builds (**8.5.7**, **8.4.22**, **8.3.31**, **8.2.31**) remain listed so existing installs stay manageable (site selectors, uninstall, extensions).
- **Series field** — each PHP package now declares a `series` (e.g. `8.4`) so later releases can offer same-series upgrades such as 8.4.22 → 8.4.23.
- **Archive fallback** — every PHP entry includes an `archives/` mirror, and the installer also derives that URL automatically if a future build leaves the main releases folder.

### Mailpit

- **No false startup error** — if Mailpit is enabled for auto-start but not installed yet, Stackroot skips quietly and shows a neutral “not installed” state instead of treating it as a failed start.

### Runtime location

- **Tools moved to Local AppData** — downloaded runtimes (PHP, Node, nginx, shims, browsers, …) now live under `%LOCALAPPDATA%\Stackroot\runtime` instead of Roaming AppData.
- **Automatic migration** — existing installs are copied and package paths are rewritten on first launch. Settings, sites, and logs stay under `%APPDATA%\Stackroot`.
- **Uninstall cleanup** — removing Stackroot clears both old and new Stackroot `bin` entries from your user PATH (data folders are left alone).

### Upgrading from 0.3.5

Your sites, databases, and settings are preserved. No manual steps required. On first launch after upgrade, Stackroot may take a moment to migrate the runtime folder.
